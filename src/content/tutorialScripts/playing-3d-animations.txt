Playing a 3D animation means telling a model to run one of the animation clips that comes with it.
In this example, s is a 3D model of type Sinbad. Sinbad comes with several bundled animations, such as Dance and other character actions.
[video: /assets/tutorials/playing-3d-animations/sinbad-dance-regular-loop.mp4?v=1]
The command s.Dance tells the model named s to play its Dance animation at regular speed. Regular speed means the animation plays at its normal built-in timing.
If you want to control the speed, add a speed value to the animation command.
[video: /assets/tutorials/playing-3d-animations/sinbad-dance-fast-loop.mp4?v=1]
For example, s.Dance at speed of 2 plays the same Dance animation faster than normal.
A higher speed value makes the animation faster, and a lower speed value makes it slower. This is useful when the same animation needs to feel calm, dramatic, or fast during gameplay.
