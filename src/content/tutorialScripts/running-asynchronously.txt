Async means the procedure starts without blocking the next command.
Without async, SceneMax waits for the procedure to finish before continuing.
Here, spin_box starts turning the box, and the move command can run at the same time.
This is useful when several actions should overlap.
Use async for visual effects, background motion, delayed UI, and fire-and-forget gameplay actions.
