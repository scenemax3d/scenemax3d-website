A go condition is a gate placed before a procedure, event, or action.
The command below the condition can run only when the condition is true.
Here, game_over is allowed only when game_status equals 0 and ko equals 1.
This protects gameplay actions from running at the wrong time.
Use go conditions for attacks, game-over screens, level transitions, and input that should work only in a specific state.
