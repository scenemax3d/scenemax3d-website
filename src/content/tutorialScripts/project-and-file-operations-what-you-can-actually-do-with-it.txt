A project is the main folder for one SceneMax game or demo. It keeps scripts, scenes, models, images, sounds, UI files, and generated files together.
Projects are needed because a game has many files. If every file is loose on the computer, it becomes hard to know what belongs to the game, what should be packaged, and what can be safely edited.
To create a project, use File, Projects, New Project. To open an existing one, use File, Projects, and select it from the project list.
Folders inside the project keep work organized. You might use folders for scripts, scenes, assets, UI, effects, and tests. Create a folder under the project when a group of files has the same job.
To add a script file, choose the folder where it belongs and create the new script there. To edit an existing file, double-click it in the project browser; SceneMax opens an editor tab for that file.
Files are saved when you press Control plus S, and the current script is also saved when you run it with F8.
Rename a file when its name no longer explains its job. Move a file when it belongs in a better folder. Delete a file only when nothing in the project still uses it.
The benefit of project and file operations is control: you always know where the game files are, what each file is for, and how to change the right file without touching the wrong one.
